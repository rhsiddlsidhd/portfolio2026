# 1. 프로젝트 개요 (Project Overview)

## 1.1. 목적 (Purpose)

이 포트폴리오는 저의 프론트엔드 개발 역량을 효과적으로 보여주기 위한 웹 애플리케이션입니다. 개인 프로젝트 및 참여 프로젝트를 소개하고, 제가 다룰 수 있는 기술 스택을 명확히 제시하여 채용 담당자 및 잠재적 협력자에게 저의 강점과 잠재력을 어필하는 것을 목표로 합니다.

## 1.2. 주요 기술 스택 (Key Technology Stack)

- **언어**: TypeScript
- **프레임워크/라이브러리**: React
- **스타일링**: Tailwind CSS
- **상태 관리**: Context API
- **데이터 페칭**: axios (인터셉터를 통한 에러 핸들링 포함)
- **번들러**: Vite

## 1.3. 목표 및 비전 (Goals & Vision)

- **개인 브랜딩 강화**: 저의 개발 철학과 문제 해결 방식을 효과적으로 전달하여 개인 브랜딩을 강화합니다.
- **기술 역량 시연**: TypeScript, React, Tailwind CSS 등 주요 기술 스택에 대한 깊은 이해와 활용 능력을 실제 작동하는 프로젝트를 통해 보여줍니다.
- **커뮤니케이션 도구**: 면접 시 저의 프로젝트와 기술에 대해 설명할 수 있는 효과적인 시각 자료 및 대화의 기반을 제공합니다.
- **최신 웹 트렌드 반영**: 성능 최적화, 반응형 디자인 등 최신 웹 개발 트렌드를 반영하여 사용자 경험을 향상시킵니다.

## 1.4. 주요 기능 (Key Features)

- **프로젝트 쇼케이스**: 각 프로젝트에 대한 상세 설명, 사용 기술 스택, 기여도, 라이브 데모 링크 및 GitHub 리포지토리 링크를 제공합니다.
- **기술 스택 페이지**: 제가 숙련된 기술들을 카테고리별로 분류하여 보여주고, 각 기술에 대한 저의 경험과 이해도를 간략히 설명합니다.
- **자기소개 및 경험**: 저의 개발 경력, 교육 배경, 핵심 역량 등을 요약하여 보여줍니다.
- **연락처 및 소셜 미디어**: 방문자가 저에게 연락할 수 있는 다양한 방법(이메일, LinkedIn, GitHub 등)을 제공합니다.
- **반응형 디자인**: 모든 디바이스(데스크톱, 태블릿, 모바일)에서 최적의 사용자 경험을 제공합니다.

## 1.5. 타겟 사용자 (Target Audience)

- **채용 담당자 및 리크루터**: 저의 기술 역량과 프로젝트 경험을 빠르게 파악하고자 하는 인사 담당자.
- **프론트엔드 개발자**: 저의 코드 스타일, 기술 선택, 프로젝트 구조에 관심 있는 동료 개발자.
- **잠재적 협력자**: 함께 프로젝트를 진행할 파트너 또는 팀원.

## 1.6. 배포 전략 (Deployment Strategy)

- **호스팅 플랫폼**: Vercel 또는 Netlify를 사용하여 빠르고 안정적인 배포를 진행합니다.
- **CI/CD**: GitHub Actions를 활용하여 메인 브랜치에 푸시될 때마다 자동으로 빌드 및 배포되도록 설정합니다.
- **도메인**: 개인 도메인을 연결하여 접근성을 높이고 전문성을 강조합니다.
